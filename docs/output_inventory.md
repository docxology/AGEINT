# Output Inventory - AGEINT generated artifacts and downstream validators

Every file under `output/` is disposable but regeneratable. This document
records what `src/build_pipeline.run_build()` produces and which downstream
stages consume it. It lives under `docs/` because generated trees are not
primary authoring surfaces.

## Build command: refresh data, variables, figures, and manuscript

```bash
uv run python scripts/build_curriculum.py
```

Canonical orchestration: `src/build_pipeline.py::run_build()`.

**Placeholder figures:** default builds can write deterministic text-plate PNGs
when Mermaid/Chrome is unavailable, but the current registry should contain no
placeholder plates after a normal rebuild. Set `AGEINT_REQUIRE_RENDERED_FIGURES=1`
to fail instead of falling back. See [`src/figures/AGENTS.md`](../src/figures/AGENTS.md).

When AGEINT runs through the full template pipeline, `output/data/curriculum_outline.json`
and `output/data/manuscript_variables.json` should appear in the project's
declared artifact set so stage validation does not flag them as undeclared.

## Regeneration sequence: full build, figures, strict renders, and hooks

```bash
# Full build (data, variables, BibTeX, figures, manuscript)
uv run python scripts/build_curriculum.py

# Figure-only pass (when curriculum data already exists)
uv run python scripts/generate_figures.py

# Require full Mermaid/Chrome renders (no placeholder PNGs)
AGEINT_REQUIRE_RENDERED_FIGURES=1 uv run python scripts/build_curriculum.py

# Output doc scaffolding after a clean stage
uv run python scripts/setup_hook.py

# Template compatibility entry (delegates to full build)
uv run python scripts/z_generate_manuscript_variables.py
```

## Adding a new output artifact: source implementation, manifest row, and tests

1. Implement generation in `src/` (not in `scripts/`).
2. Wire the call from `src/build_pipeline.py::run_build()` when it belongs in the default build.
3. Append a row to the table above.
4. If the artifact is consumed by the template pipeline validator, declare it in the project output contract.
5. Re-run `uv run pytest tests/ --cov=src --cov-fail-under=90` and rebuild.
